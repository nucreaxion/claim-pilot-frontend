Performance note: The current pipeline is sequential (Planner → Policy → Fraud → Answer). Policy and Fraud agents could run in parallel since they are independent, reducing latency by ~50% for those steps. This optimization is planned when migrating to LangGraph StateGraph.

Agent Pipeline — Internal Data Flow

How the four AI agents pass state through the workflow, what each agent produces, and how the final response is assembled from intermediate results.

Agent State Machine

Each agent receives accumulated state and enriches it
stateDiagram-v2
    [*] --> PlannerAgent: Input State
{ claim_description, policy_id,
claim_id, question }

    PlannerAgent --> PolicyAgent: Plan
{ needs_policy_search: true,
needs_fraud_check: true,
search_query: "..." }

    PolicyAgent --> FraudAgent: Policy Result
{ decision, confidence,
reasoning, clauses[] }

    FraudAgent --> AnswerAgent: Fraud Result
{ fraud_score: 0.27,
signals[] }

    AnswerAgent --> [*]: Final Response
{ decision, confidence,
reasoning, citations[],
fraud_score, next_action }

    note right of PlannerAgent
        Pure logic - no external calls
        Determines which tools to invoke
    end note

    note right of PolicyAgent
        Calls MCP: /tools/policy-search
        Analyzes clause text for keywords
        Returns coverage decision
    end note

    note right of FraudAgent
        Calls MCP: /tools/fraud-score
        Skips if no claim_id provided
        Returns risk assessment
    end note

    note right of AnswerAgent
        Pure logic - no external calls
        Transforms clauses into citations
        Composes final structured response
    end note

Data Models & Schemas

Pydantic models that define the API contracts between services. Each service owns its own copy of these schemas to allow independent evolution.

Entity Relationship — API Contracts

classDiagram
    class CoverageCheckRequest {
        +str claim_description [min_length=10]
        +str policy_id
        +str|None customer_id
        +str|None claim_id
        +str question = "Is this claim covered?"
    }

    class CoverageCheckResponse {
        +Literal decision
        +float confidence
        +str reasoning
        +list~Citation~ citations
        +float|None fraud_score
        +str next_action
    }

    class Citation {
        +str document
        +int|None page
        +str|None clause_title
        +str|None text
    }

    class PolicyClause {
        +str policy_id
        +str document
        +int page
        +str clause_title
        +str clause_text
        +float confidence
    }

    class FraudResult {
        +str claim_id
        +float fraud_score
        +list~str~ signals
    }

    class ClaimDetails {
        +str claim_id
        +str claim_type
        +int amount
        +str status
    }

    CoverageCheckResponse "1" *-- "0..*" Citation
    CoverageCheckRequest ..> CoverageCheckResponse : produces
    PolicyClause ..> Citation : transformed to
    FraudResult ..> CoverageCheckResponse : fraud_score
    ClaimDetails ..> FraudResult : input

    note for CoverageCheckResponse "decision: covered | not_covered\n| needs_review | unknown"
